# Playability Definition

## Goal
Optimize for a **fully playable supported slice**, rather than attempting to support all BioWare games at once.

## Target Slice Progression

### Milestone A: KotOR I Endar Spire
- **Requirements:** 
  - Fully completable
  - No crashes
  - No softlocks

### Milestone B: Taris Entry Slice
- **Requirements:**
  - Fully completable
  - No crashes
  - No script halts

### Future Milestones
Only after achieving the milestones above will we expand parity outward. This aligns with the repository's current strategy (Endar Spire first, then early Taris), preventing the pitfalls of trying to solve all four games simultaneously.

## Development Rules
- **Rule 1:** The AI coder is **forbidden** from adding nonessential features during the stabilization of these milestones.
- **Rule 2:** There is only one official "supported slice" at any given time.
- **Rule 3:** Every bug must map to a specific severity and system constraint.
